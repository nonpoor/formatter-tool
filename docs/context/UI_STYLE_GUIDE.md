# UI_STYLE_GUIDE

## 目标
确保后续新增界面、模块或交互时，视觉风格与当前产品保持一致，避免“越做越散”。

## 视觉基线
1. 风格：简约、轻玻璃感、偏苹果式克制动效。
2. 颜色：使用 `globals.css` 与 `FormatterTool.module.css` 中已存在变量，不随意新增主色。
3. 字体：沿用现有字体栈（`SF Pro` + `PingFang SC`）。
4. 圆角与阴影：延续当前 panel/button 的圆角与阴影层级，不做强对比重阴影。

## 组件规则
1. 按钮只保留三类语义：
   - 主操作（primary）
   - 次操作（secondary）
   - 辅助操作（ghost）
2. 状态反馈统一：
   - 成功：绿色系
   - 失败：红色系
   - 中性：灰蓝系
3. 新增控件优先复用已有 class 语义，避免重复定义近似样式。

## 布局规则
1. 主页面容器宽度沿用：`min(1120px, 92vw)`。
2. 内容块优先使用卡片式 panel，保持统一间距节奏（14/18/24）。
3. 移动端适配必须同步验证，保持当前断点行为（`max-width: 780px`）。

## 动效规则
1. 动效以“低频高价值”为主：页面入场、按钮 hover/active、状态变化。
2. 时长建议：`150ms ~ 550ms`；禁止明显拖沓动效。
3. 禁止引入炫技动画（连续闪烁、大幅位移、长链条过渡）。

## 文案与可用性
1. 所有关键操作必须有可见反馈（按钮文案、状态栏、`aria-live`）。
2. 控制台/状态文案优先简短中文，语义明确（成功/失败/处理中）。
3. 可复制、可导出、可清空等行为必须保持一致反馈模式。

## 新功能接入流程（强制）
1. 新增 UI 前先写一句“与现有风格一致的设计意图”。
2. 修改后至少检查：
   - Desktop + Mobile 显示
   - 按钮状态反馈
   - 颜色/间距是否偏离当前基线
3. 若出现新设计语言（如模板上传页）：
   - 先更新此文档，再实施代码。
