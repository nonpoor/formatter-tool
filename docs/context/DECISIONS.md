## 2026-04-04：MVP 收敛为 `general | academic` 双模式
- 决策：前端与主流程只保留 `general`、`academic` 两个模式入口，不再暴露多模板。
- 背景：模板维度过多会提升用户决策成本，并快速拉高回归复杂度。
- 为什么这样做：先把“规范排版 + 目录友好结构”主链路做稳，再考虑后续扩展。
- 放弃方案：继续保留实验报告/课程论文/通用作业等细分模板入口。
- 状态：已落地；类型由 `TemplateId` 收敛为 `ModeId`。

## 2026-04-04：数学能力表述收敛为“文本保真”，不宣称渲染支持
- 决策：当前阶段只承诺 LaTeX 常见定界符的“文本保真”，不承诺网页或 docx 的数学公式对象渲染。
- 覆盖范围（稳妥优先）：`$...$`、`$$...$$`、`\\(...\\)`、`\\[...\\]`。
- 为什么这样做：先确保 normalize / preview / clipboard / docx 链路不误清洗、不误拆分、不误破坏，再评估重型公式渲染。
- 关联实现：
  - `src/features/formatter/math/preservation.ts`：集中公式占位保护（非贪心、保守匹配）。
  - `src/features/formatter/parser/markdownParser.ts` + `src/features/formatter/normalize/index.ts`：接入保护与恢复。
  - `DocumentMeta.math`：暴露 `detected/spanCount/protectionApplied` 供调试与验收。
- 状态：已落地（含自动化回归与开发验收页）。

## 2026-04-04：新增半自动验收页，优先降低人工找样本成本
- 决策：新增开发用验收页与固定样本库，覆盖“数学公式保真 + 学术模式边界”高风险场景。
- 入口：`/dev/formatter-acceptance`。
- 页面能力：样本切换、模式切换、预览、clipboard html/text、doc 结构摘要、导出 docx、math 检测统计。
- 为什么这样做：把重复性验证前置到固定样本与自动化回归，只保留少量必须在 WPS/Word 人工确认的项目。
- 状态：已落地。

## 2026-04-04：分项表达规范化策略（替代“无序列表全量处理”）
- 决策：分项表达按语义分流，不再把无序列表当作单一处置对象。
- 规范化策略命名：
  - `explicit_step_only`：仅显式“步骤/流程/实验过程/方法步骤”等语义触发。
  - `step_or_sequence`：显式步骤语义或明显时序词（如“首先/然后/最后”）触发。
- 背景：普通要点总结、注意事项、材料清单等并不应该被强制编号。
- 为什么这样做：减少误转换，保持内容原意与可读性。
- 放弃方案：学术模式下把全部无序列表统一转有序。
- 状态：已落地。

## 2026-04-04：学术模式新增“解释型项目符号 -> 分项正文段落”
- 决策：学术模式下，非步骤型无序项目符号默认去黑点并规范为分项正文段落。
- 约束：
  - 保留“关键词 + 解释”的层次结构（包括加粗关键词和冒号分界）。
  - 清单型短项也去黑点，但保持独立分段，不强制编号。
  - 输出保持“每项独立段落”，不合并成连续长段。
- MVP 主动取舍：
  - 嵌套黑点/空心点脏输入当前优先扁平化提可读性，不强求完整层级保真。
- 为什么这样做：让学术模式更接近正式论文/实验报告正文表达，而不是网页笔记式项目符号。
- 状态：已落地（通过回归测试覆盖）。

## 2026-04-04：默认策略集中配置到 `config/policies.ts`
- 决策：normalize/paste/docx 默认行为统一由 `src/features/formatter/config/policies.ts` 提供。
- 背景：历史默认值分散在多个模块，维护和追踪困难。
- 为什么这样做：降低策略漂移与行为不一致风险，便于后续回归核对。
- 放弃方案：维持“各模块各自默认值”。
- 状态：已落地。

## 2026-04-04：clipboard 职责命名拆分
- 决策：渲染函数统一为 `renderClipboardPayload`，写入函数统一为 `writeClipboardPayload`，`copyOptimizedToClipboard` 保留为薄封装。
- 背景：历史命名把“渲染 payload”与“浏览器写入”混在同一语义里。
- 为什么这样做：明确职责边界，降低后续维护理解成本。
- 状态：已落地。

## 2026-04-04：WPS 目标表述保持克制
- 决策：对内对外统一表述为“更适合 WPS 标题样式映射与目录生成的结构”。
- 背景：不同环境、粘贴选项与客户端版本存在差异，无法承诺 100% 自动目录结果。
- 为什么这样做：降低过度承诺风险，保持预期管理准确。
- 状态：已落地（文案已同步收敛）。

## 2026-04-03：引入结构模式 `legacy | academic`
- 决策：在模板配置引入 `structureMode`，由模板驱动 normalize 行为。
- 背景：同一套规则难以同时满足“通用兼容”和“学术目录友好”。
- 为什么这样做：把高强度规则限制在学术模板，降低默认模式回归风险。
- 放弃方案：全局统一强规则（风险过高，误判影响普通场景）。
- 状态：已落地，后续可扩展模板级覆盖。

## 2026-04-03：学术模式统一将无序列表转有序列表
- 决策：学术模板下，将无序项目符号统一成有序子项。
- 背景：用户明确希望论文/作业格式更接近可编号结构，不再执着圆点保留。
- 为什么这样做：目录友好、结构一致、便于教师阅读与后续样式调整。
- 放弃方案：保留原圆点（会导致风格不统一且和目标冲突）。
- 状态：已实现，保留嵌套层级。

## 2026-04-03：标题识别采用“保守启发式”
- 决策：仅识别有限前缀（如 `1.`/`1、`/`一、`/`1.1`/`（一）`/`1.1.1`），并要求短标题文本。
- 背景：单段粘贴时正文句子容易被误判为标题。
- 为什么这样做：宁可漏判少量标题，也要优先降低误判对全文结构的破坏。
- 放弃方案：激进识别（误判率不可控）。
- 暂时性：阈值（长度/标点规则）未来可能依据样本调整。

## 2026-04-03：粘贴策略改为“结构优先兜底”
- 决策：默认仍优先 `text/plain`，但若 HTML 结构评分显著更优，则采用 HTML 投影纯文本。
- 背景：同内容在“整段复制”和“抽段复制”时，plain 常丢失列表层级。
- 为什么这样做：在不强制富文本粘贴的前提下，提升结构恢复概率。
- 放弃方案：始终 plain 或始终 html（前者损结构，后者侵入性强）。
- 状态：已落地，阈值策略为 `htmlScore >= plainScore + 2`。

## 2026-04-03：不自动插入 Word TOC 字段
- 决策：当前只保证 Heading 1-3 样式正确，不自动写入 TOC field。
- 背景：TOC 字段兼容性、更新行为、用户预期差异较大。
- 为什么这样做：先确保结构正确与稳定，再考虑自动目录功能。
- 放弃方案：本阶段直接自动插 TOC（复杂度与风险不匹配）。
- 状态：长期候选，不在当前里程碑内。
